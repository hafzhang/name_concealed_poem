# 三、用户流程（优化版 - 游客优先 + 防刷机制）

为了降低门槛，采用 **"先尝后买，用完即走"** 的策略，**完全不需要强制登录**。

```mermaid
graph TD
    A[首页] --> B(输入姓名 + 选择风格)
    B --> C[点击生成]
    C --> D{游客额度检查}
    D -- 有剩余(默认1次/天) --> E[生成诗句 + 预览]
    D -- 次数用尽 --> F{提示解锁方式}
    F --> G[登录 +3次]
    F --> H[邀请好友 +1次]
    F --> I[看广告 +1次]
    F --> J[付费解锁无限次]
    E --> K[生成图片]
    K --> L[保存到本地]
    L -.-> M[引导登录: "登录后永久保存记录 + 获得更多次数"]
```

> **核心策略：多层防刷机制**
>
> 1. **游客模式限制**
>    *   设备指纹 (LocalStorage + Canvas Fingerprint) 生成 `Guest ID`
>    *   **每日限制**：每设备每天 1 次免费（而不是 3 次，降低刷量价值）
>    *   **IP 限制**：同一 IP 每天最多 10 次总访问（防止代理 IP 刷）
>    *   清除缓存无效：指纹基于 Canvas 渲染特征，不依赖 Cookie
>
> 2. **注册用户奖励**
>    *   手机号验证（短信验证码）：+3 次生成额度
>    *   微信一键登录：+3 次生成额度
>    *   邀请好友：双方各 +1 次（无限裂变）
>
> 3. **防刷技术手段**
>    *   前端：设备指纹 + 行为分析（检测自动化脚本）
>    *   后端：Rate Limiting（每分钟最多 5 次请求）
>    *   数据库：标记异常用户（高频请求、同一设备多账号）

# 四、技术实现方案（2025 极速开发版）

## 核心技术栈（推荐）

*   **框架**: Next.js 14+ (App Router) - 全栈能力，部署方便 (Vercel)
*   **UI 组件**: Tailwind CSS + Shadcn/ui - 快速构建优美界面
*   **图片生成**: **Satori + @resvg/resvg-js**
    *   *原理*：用 HTML/CSS 写排版，自动转 SVG 再转 PNG。
    *   *优势*：比 Canvas/Pillow 更容易调试布局，支持复杂字体和样式。
    *   *字体优化*: 使用 `fonttools` 进行字体子集化（仅保留常用汉字，从 10MB+ 降至 2MB）
*   **数据库**: Vercel Postgres / Supabase (存储生成记录、用户积分)
*   **AI 模型**:
    *   DeepSeek-V3 / GPT-4o-mini (高性价比，中文理解力强)
*   **缓存层**: Redis (Vercel KV) - 缓存高频名字的生成结果，减少 AI 调用
*   **队列系统**: Vercel Cron / Bull Queue - 处理并发生成请求，避免 API 限流

## 核心流程伪代码

### 方案 A：快速模式（默认，适合急躁用户）
1.  **一键生成**
    *   用户提交姓名 + 风格选择
    *   后端先查缓存（Redis）-> 命中则直接返回（< 100ms）
    *   未命中则调用 AI API -> 返回 JSON 诗句
    *   立即触发图片渲染（Satori）-> 返回图片 URL
    *   *耗时：AI 调用 2-3s + 渲染 500ms*

### 方案 B：分步模式（适合挑剔用户）
1.  **Step 1: 文本生成**
    *   用户提交姓名 -> API 调用 LLM -> 返回 JSON 诗句 -> **前端展示文本**。
    *   *此时用户可以点击"换一首"（免费，不扣次数）或直接修改文字。*
2.  **Step 2: 图片合成**
    *   用户确认无误 -> 前端将最终诗句 + 风格配置发送给后端 -> Satori 渲染 HTML -> 转 PNG。
3.  **Step 3: 交付**
    *   返回图片 URL -> 存入数据库 -> 用户下载/分享。

### 错误处理机制
```javascript
// 伪代码示例
async function generatePoem(name, style) {
  // 1. 检查缓存
  const cached = await redis.get(`poem:${name}:${style}`);
  if (cached) return JSON.parse(cached);

  // 2. 调用 AI，带重试
  let attempts = 0;
  while (attempts < 3) {
    try {
      const result = await aiClient.generate(prompt);
      // 3. 验证藏头格式
      if (validateAcrostic(result.poem, name)) {
        await redis.set(`poem:${name}:${style}`, JSON.stringify(result), 'EX', 86400);
        return result;
      }
    } catch (error) {
      attempts++;
      if (attempts === 3) {
        // 返回降级方案（预设模板）
        return getFallbackTemplate(name);
      }
    }
  }
}
```

# 六、数据模型 (Schema Design)

> **为什么必须存数据？**
> 1.  **计费核心**: 必须记录用户的剩余次数 (`credits`) 和会员状态 (`is_vip`)，否则无法变现。
> 2.  **降低 AI 成本 (缓存)**: 当用户重复输入同一个名字时，直接从数据库调取历史结果，**省去一次 API 调用费用**。
> 3.  **用户留存**: 允许用户在“我的作品”中找回之前的生成记录。

使用 Supabase / Postgres 的简单设计：

### 1. Users 表 (用户)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | UUID | 主键 |
| openid | String | 微信/第三方 ID |
| credits | Integer | 剩余生成次数 (默认 3) |
| is_vip | Boolean | 是否付费用户 |

### 2. Poems 表 (生成记录)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | UUID | 主键 |
| user_id | UUID | 关联用户 |
| input_name | String | 输入的姓名 |
| poem_content | JSONB | 诗句内容 |
| style_config | JSONB | 风格配置 (字体/背景) |
| image_url | String | 生成图片的存储地址 (Nullable) |
| status | String | 'draft' (仅文本) / 'completed' (已出图) |
| created_at | Timestamp | 生成时间 |

# 九、内容风控与合规

## 多层内容审核机制

### 第 1 层：输入层过滤（阻断违规输入）
```javascript
// 敏感词库（持续更新）
const BLACKLIST = {
  political: ['习近平', '毛泽东', '...', ],  // 政治人物
  offensive: ['傻逼', '操你', '...', ],       // 辱骂词汇
  religious: ['耶稣', '真主', '佛祖', ],       // 宗教领袖
  superstition: ['算命', '改运', '风水', ],   // 封建迷信
}

// 校验逻辑
function validateName(name) {
  for (let category in BLACKLIST) {
    if (BLACKLIST[category].some(word => name.includes(word))) {
      return {
        valid: false,
        reason: '该名称暂不支持生成，请更换其他名字'
      }
    }
  }
  return { valid: true }
}
```

### 第 2 层：生成层约束（AI Prompt 限制）
在 Prompt 中明确禁止：
```
禁止事项：
1. 不得涉及政治、宗教、时政敏感话题
2. 不得使用封建迷信、命理算命的表述
3. 不得生成低俗、暴力、歧视性内容
4. 不得对人物进行评价或预测（如"富贵""长寿"等）
5. 必须明确标注：本作品仅为文化创作，不含任何预测性质
```

### 第 3 层：输出层审核（高风险内容人工复核）
* 标记高风险输入（如政治相关词汇）
* 触发人工审核队列
* 审核通过后才生成图片

### 第 4 层：用户举报机制
* 每张生成结果下方有"举报"按钮
* 举报理由：违规内容 / 不当言论 / 其他
* 被举报作品自动下架，等待审核

* **不可去除**：即使用户付费去水印，AI 标识也必须保留

```mermaid
    J -- 通过 --> I
    J -- 拒绝 --> K[通知用户 + 驳回]
```

## 运营风控

### 1. 数据监控指标
* **违规率**：违规生成 / 总生成（应 < 1%）
* **举报率**：被举报作品 / 总作品（应 < 0.5%）
* **高危词排行**：每周统计 Top 10 高频触发词

### 2. 应急响应机制
* **发现违规内容**：立即下架 + 用户封禁
* **大量恶意输入**：触发 IP 封禁 + 验证码
* **监管要求**：预留内容删除接口，支持根据监管要求删除指定内容

# 十二、数据分析体系（数据驱动优化）

## 核心指标定义

### 北极星指标
**付费转化率**：从免费用户到付费用户的转化比例
* 目标值：> 5%（工具类产品平均水平）
* 计算公式：付费用户数 / 总注册用户数

### 一级指标（健康度）
| 指标 | 定义 | 目标值 |
|:---|:---|:---:|
| DAU | 日活跃用户数 | 持续增长 |
| 留存率（次日） | 第二天还来使用的比例 | > 30% |
| 留存率（7日） | 第 7 天还来使用的比例 | > 15% |
| ARPU | 每用户平均收入 | > ¥2 |
| 付费率 | 付费用户占比 | > 5% |

### 二级指标（优化方向）
| 指标 | 定义 | 优化方向 |
|:---|:---|:---|
| 生成成功率 | 成功生成诗句 / 尝试次数 | AI Prompt 优化 |
| 分享率 | 分享次数 / 生成次数 | 分享 incentives |
| 低清→高清转化 | 购买高清 / 低清预览 | 定价策略 A/B 测试 |
| 邀请成功率 | 完成邀请 / 发起邀请 | 邀请流程优化 |
| 广告完成率 | 看完广告 / 点击广告 | 广告时长优化 |

## 数据埋点方案

### 关键埋点事件
```javascript
// 1. 用户行为
track('page_view', { page: 'home' })
track('button_click', { button: 'generate' })

// 2. 生成流程
track('generate_start', { name: '张三', style: 'ruya' })
track('generate_success', { cached: true, duration: 2500 })
track('generate_fail', { reason: 'ai_timeout' })

// 3. 付费转化
track('purchase_view', { package: 'basic_5' })
track('purchase_click', { package: 'basic_5', price: 9.9 })
track('purchase_success', { amount: 9.9, method: 'wechat' })

// 4. 分享裂变
track('share_click', { platform: 'wechat' })
track('share_success', { platform: 'wechat', poem_id: 123 })
track('invite_accept', { inviter_id: 456 })
```

### 数据可视化
* **实时看板**：今日 DAU、收入、生成次数
* **漏斗分析**：访问 → 输入 → 生成 → 预览 → 付费
* **用户分群**：免费用户 / 付费用户 / 高价值用户（复购 > 3 次）

# 十四、成本与收益预估

## 成本结构（月度）

### 固定成本
| 项目 | 金额 | 说明 |
|:---|:---:|:---|
| 域名 | ¥10/年 | .com 域名 |
| 服务器 | ¥0 | Vercel 免费版（初期） |
| 数据库 | ¥0 | Supabase 免费版（< 500MB） |
| **合计** | **¥0** | MVP 阶段几乎零成本 |

### 变动成本（随用户增长）
| 项目 | 单价 | 说明 |
|:---|:---:|:---|
| AI API | ¥0.001/1K tokens | DeepSeek-V3（约 ¥0.02/次生成） |
| 图片存储 | ¥0.1/GB | Vercel Blob / 阿里云 OSS |
| 短信验证 | ¥0.04/条 | 阿里云短信 |
| 支付手续费 | 0.6% | 微信支付费率 |

### 成本测算示例
假设月度生成 10,000 次：
* AI 成本：10,000 × ¥0.02 = ¥200
* 存储：10,000 × 0.5MB = 5GB ≈ ¥0.5
* 短信：假设 30% 注册验证 = 3,000 × ¥0.04 = ¥120
* 支付手续费：假设月收入 ¥10,000 × 0.6% = ¥60
* **总成本 ≈ ¥400**

## 收益预估（保守）

### 场景 1：月 DAU 500（第 1 个月）
* 付费转化率 5% = 25 人付费
* 客单价 ¥15（平均）
* **月收入 = 25 × ¥15 = ¥375**
* **净利润 = ¥375 - ¥20 = ¥355**

### 场景 2：月 DAU 2,000（第 3 个月）
* 付费转化率 5% = 100 人付费
* 客单价 ¥20（复购用户提升）
* **月收入 = 100 × ¥20 = ¥2,000**
* **净利润 = ¥2,000 - ¥80 = ¥1,920**

### 场景 3：月 DAU 10,000（第 6 个月）
* 付费转化率 5% = 500 人付费
* 客单价 ¥25（VIP 订阅占比提升）
* **月收入 = 500 × ¥25 = ¥12,500**
* **净利润 = ¥12,500 - ¥400 = ¥12,100**

## 盈亏平衡点

* 假设每月固定成本 ¥0（使用免费版服务）
* 变动成本约占收入 3%
* **盈亏平衡点：只要有 1 个付费用户即可盈利**

## 风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|:---|:---:|:---:|:---|
| AI 价格上涨 | 中 | 高 | 接入多个供应商（DeepSeek + GPT-4o-mini） |
| 字体版权纠纷 | 低 | 高 | 仅使用免费可商用字体 |
| 用户增长停滞 | 中 | 中 | 拓展获客渠道（付费投放、SEO） |
| 恶意刷量 | 高 | 低 | 多层防刷机制（已实现） |
| 政策监管 | 低 | 高 | 严格内容审核 + AI 标注 |
